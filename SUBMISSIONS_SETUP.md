# نظام استقبال الطلبات - Submissions System

## مميزات النظام

✅ استقبال طلبات المعاينة من الصفحة الرئيسية
✅ استقبال رسائل التواصل من صفحة التواصل
✅ حفظ الطلبات في قاعدة بيانات محلية (JSON)
✅ واجهة إدارية لعرض جميع الطلبات
✅ رسائل تأكيد عند الإرسال الناجح
✅ معالجة الأخطاء والرسائل التوضيحية

---

## كيفية الاستخدام

### 1. التحقق من الطلبات

بعد تثبيت المشروع، يمكنك عرض جميع الطلبات المستقبلة عبر الرابط:

```
http://localhost:3000/admin/submissions
```

### 2. الملفات الرئيسية

- **`/src/app/api/submissions/route.js`** - API لاستقبال وحفظ الطلبات
- **`/src/app/admin/submissions/page.js`** - لوحة تحكم لعرض الطلبات
- **`/data/submissions.json`** - قاعدة البيانات (يتم إنشاؤها تلقائياً)

### 3. طلبات المعاينة (من الصفحة الرئيسية)

تُحفظ الطلبات بالتفاصيل التالية:
- الاسم الكامل
- البريد الإلكتروني
- رقم الجوال
- الموقع/العنوان
- نوع الخدمة المطلوبة
- الملاحظات الإضافية

### 4. رسائل التواصل (من صفحة التواصل)

تُحفظ الرسائل بالتفاصيل التالية:
- الاسم الكامل
- البريد الإلكتروني
- رقم الجوال
- نص الرسالة

---

## التكامل مع خدمات البريد الإلكتروني (اختياري)

يمكنك إضافة خدمة بريد إلكتروني مثل:

### استخدام Gmail
```javascript
const nodemailer = require('nodemailer');

const transporter = nodemailer.createTransport({
  service: 'gmail',
  auth: {
    user: process.env.EMAIL_USER,
    pass: process.env.EMAIL_PASSWORD,
  }
});
```

### استخدام SendGrid
```javascript
const sgMail = require('@sendgrid/mail');
sgMail.setApiKey(process.env.SENDGRID_API_KEY);
```

### استخدام Resend (الموصى به للـ Next.js)
```javascript
import { Resend } from 'resend';
const resend = new Resend(process.env.RESEND_API_KEY);
```

## المتغيرات البيئية المطلوبة (اختياري)

إذا أردت إرسال رسائل بريد إلكتروني تلقائية:

```env
EMAIL_USER=your-email@gmail.com
EMAIL_PASSWORD=your-app-password
SENDGRID_API_KEY=your-sendgrid-key
RESEND_API_KEY=your-resend-key
```

## الخطوات التالية

1. **إضافة قاعدة بيانات حقيقية** - استبدل النظام الحالي بـ Supabase أو Neon أو MongoDB
2. **إرسال رسائل بريد إلكتروني** - أضف تنبيهات بريدية عند وصول طلب جديد
3. **حماية لوحة التحكم** - أضف نظام مصادقة للاطلاع على الطلبات
4. **تصدير البيانات** - أضف خاصية تصدير الطلبات كـ Excel أو PDF

## استكشاف الأخطاء

إذا لم تجد الطلبات:

1. تأكد من وجود مجلد `/data` في جذر المشروع
2. تحقق من رسائل الخطأ في console
3. تأكد من صلاحيات الكتابة على القرص
4. حاول إعادة تشغيل خادم التطوير
